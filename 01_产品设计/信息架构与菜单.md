# 信息架构与菜单

当前前端路由位于 `delivery-platform-web/src/router/index.ts`，信息架构如下。

## 一级菜单

| 一级菜单 | 二级页面 |
| --- | --- |
| 工作台 | 数据看板、我的待办 |
| 项目管理 | 项目台账、项目档案、文件审核、工时与日报、项目复盘 |
| 标准与知识 | 交付流程、检查模板、档案模板、文档模板、知识库、工具中心、培训记录 |
| 绩效与团队 | 目标与绩效、技能评估 |
| 组织与权限 | 组织架构、用户管理、角色权限 |
| 系统设置 | 国家配置、币种与汇率、语言与翻译、通知规则、审批配置、操作日志、系统参数、存储备份、接口集成 |

## 导航原则

- 高频执行页面放在项目管理和标准知识。
- 管理员配置集中在组织权限和系统设置。
- 知识库作为标准与知识的核心页面，支持分类、文章、附件和预览。
- 绩效与培训聚合到团队管理视角，避免散落在项目页面。

## 页面状态要求

每个列表页必须具备加载、空数据、筛选、分页和错误提示状态。每个编辑页必须具备表单校验、提交中、提交成功、提交失败和关闭确认状态。文件预览必须根据文档类型给出可读的在线查阅体验。
