# 产品方案

## 产品目标

交付管理平台用于解决跨地区软件交付中的过程不透明、档案不完整、文件审核分散、知识沉淀弱、权限边界不清和部署不可控问题。

## 核心价值

- 建立交付项目全生命周期台账。
- 将档案、文件、审核、检查清单和交付流程串成一条业务链。
- 让知识库、模板、工具和培训成为项目执行资产。
- 通过组织、角色和权限保障数据访问边界。
- 支持 Docker 本地测试和 Git 化服务器部署。

## 用户角色

- 平台管理员：维护配置、组织、用户、权限、备份和集成。
- 交付经理：管理项目组合、风险、绩效和复盘。
- 项目经理：管理项目、成员、档案、文件、日报和阶段进度。
- 实施人员：执行任务、上传文件、查阅知识库和工具。
- 审核人员：完成文件审核、档案审核和审批动作。
- 知识管理员：维护分类、文章、版本、附件和预览质量。

## 模块边界

| 模块 | 主要能力 | 不包含 |
| --- | --- | --- |
| 工作台 | 看板、待办、风险、关键指标 | 复杂 BI 建模 |
| 项目管理 | 项目、成员、款项、阶段、日报、复盘 | 财务总账 |
| 档案文件 | 档案模板、项目档案、上传、审核、版本 | 大型网盘 |
| 标准知识 | 流程、检查模板、文档模板、知识库、工具 | 外部 LMS 全量能力 |
| 绩效团队 | OKR、技能、培训参与 | 薪酬核算 |
| 组织权限 | 部门、用户、角色、权限 | 企业 AD 全量替代 |
| 系统设置 | 国家、币种、语言、通知、审批、日志、备份、集成 | 云平台控制台 |

## 当前版本范围

当前工程为本地可测试版本，重点是完成端到端功能骨架、标准目录、Arco 组件迁移、知识库预览测试、Docker 配置校验和 GitHub 推送部署流程。
