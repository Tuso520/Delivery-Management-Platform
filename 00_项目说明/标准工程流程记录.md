# 标准工程流程记录

## 执行范围

本次按软件公司标准工程流程对交付管理平台重新梳理，覆盖产品、需求、UI/UX、技术、代码、测试、审查、安全、部署、交付和复盘。

## 流程结果

1. 产品设计：明确平台面向交付中心，核心是项目交付过程、知识标准和权限治理。
2. 需求分析：按工作台、项目管理、标准知识、绩效团队、组织权限、系统设置拆分功能和验收口径。
3. UI/UX 设计：统一 Arco Design Vue，修复选择框、标题、按钮等乱码文字。
4. 技术设计：确认 REST API、JWT/RBAC、Prisma 数据模型、MinIO 文件存储、统一响应格式。
5. 代码实现：保留当前前后端源码结构，清理旧文档入口，补充标准文档。
6. 测试验收：执行前后端类型检查、测试、构建、Docker Compose 配置校验和本地模拟服务验证。
7. 代码审查：检查旧组件库口径、乱码、冗余文档、部署数据保护风险。
8. 安全性能：确认鉴权、日志、文件预览、数据保护、备份和回滚约束。
9. 部署运维：采用 GitHub 推送 + 服务器 Git 拉取 + Docker Compose 部署。
10. 最终交付：更新交付清单，提交并推送到 GitHub。
11. 复盘归档：记录标准化后续改进项。
