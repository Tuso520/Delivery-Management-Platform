# 安全说明

## 支持范围

当前只维护 `main` 分支的安全修复和部署安全说明。

## 敏感数据

禁止提交以下内容：

- `.env` 文件或生产环境变量。
- GitHub Token、SSH 密码、数据库密码、MinIO 密钥和 JWT 密钥。
- 生产数据库备份、MinIO 归档和服务器日志。
- 暴露客户数据的浏览器截图。
- 本地生成的发布包、测试产物和临时日志。

## 部署安全

生产更新必须使用 `deploy-git.sh` 或经过审查的等效流程，并满足：

- 数据库迁移前备份 MySQL。
- 容器切换前备份 MinIO，并保证数据库与对象存储按同一版本成对恢复。
- 保留服务器本地 `.env`，不从 Git 覆盖。
- 生产环境只使用 `prisma migrate deploy`。
- 种子脚本和目标数据迁移必须幂等，禁止清空业务表；删除列前必须执行 preflight、迁移审计和严格关联校验。
- API、File Worker 和 Outbox Worker 必须使用同一发布产物；集成 Secret 迁移容器、API 与 Outbox Worker 必须使用同一独立加密密钥。
- 文件下载、预览、合同/财务读取、权限变更、密码操作和高风险失败必须保留审计记录。

## 问题报告

内部发现安全问题时，应提交给交付管理平台维护人，并说明：

- 受影响模块。
- 复现步骤。
- 预期行为和实际行为。
- 影响范围。
- 可选的缓解建议。
