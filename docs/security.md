# 安全说明

## 密钥与敏感信息

禁止提交：

- `.env`
- JWT 密钥
- 数据库账号密码
- MinIO 账号密码
- SSH 凭据
- GitHub Token
- 生产备份归档

## 数据保护

- 生产部署必须在迁移前备份 MySQL 和 MinIO。
- 文件上传审批通过前，不能覆盖当前已审批文件版本。
- 被驳回的上传文件必须保持非当前版本。
- 新文件审批通过后，旧的当前版本要自动失效。
- 文件实体只存 MinIO；数据库保存逻辑文件、不可变版本、对象索引、校验和、处理状态和审计元数据。
- 上传同时执行扩展名、大小、档案项命名规则和文件签名校验；系统可编辑上限不能突破 `FILE_UPLOAD_HARD_LIMIT_MB` 部署硬上限。

## 访问控制

- 前端菜单、路由和动作按权限码展示，后端执行最终校验；角色只作为权限集合，业务 Controller 不维护角色白名单。
- 后端接口必须执行认证和权限校验。
- 项目数据默认按项目成员关系限制，高权限角色除外。
- 合同、金额、回款、待审文件和历史版本还必须执行字段或对象级权限裁剪。
- 路由权限不足只跳转首个可访问页面或无权限页，不得因只有设置权限而清除有效会话。
- 项目常规删除语义是软归档；物理删除除 `project:delete` 外还要求 `SUPER_ADMIN`，文件、审核、财务或既有审计任一存在即拒绝，成功和失败尝试都记录审计。

## 文件预览安全

- 全平台预览只读，后端和前端不提供 Office 编辑模式或保存回调。
- 预览链接使用短时签名链接或登录态鉴权，不返回 MinIO bucket、object key 或存储密钥。
- Markdown 原始 HTML 必须转义并限制链接协议；XMind 文本输出转义。页面不得重新引入任意后端 HTML 的通用 `v-html` 预览链路。
- 不支持的文件类型应回退到下载，不尝试执行文件内容。
- 浏览器预览不能暴露对象存储密钥。

## 会话、集成与并发

- Access Token 只保存在内存；Refresh Token 使用 HttpOnly Cookie，服务端只存哈希会话并支持撤销。
- 登录失败次数、锁定窗口和会话时长从系统配置读取，Redis 只保存短期计数，不保存明文凭据。
- 飞书/企业微信 Secret 使用独立 32 字节 Base64 密钥做 AES-256-GCM 加密；接口、日志、同步结果和审计只显示掩码或稳定错误码。
- 项目 revision、审核活动唯一键、数据库行锁/条件更新、同步租约和投递幂等键共同防止重复写入。409 不应由前端自动重放。
- Outbox 与通知回执不保存外部错误正文中的 Secret；达到重试上限保留 `DEAD` 事实供人工处理。

## 运维安全

- 不要手动删除 Docker 命名卷。
- 不要在未确认维护窗口和备份目标的情况下恢复数据库。
- `.env` 只保存在服务器本地，不进入 Git。
- 迁移前同时备份 MySQL 和 MinIO，回滚也必须成对恢复；已发送的外部消息不能通过数据库回滚撤回。
