# 产品说明

## 产品定位

交付管理平台是面向跨国家、跨专业交付团队的内部运营系统。它把项目概览、项目档案、统一文件审核、标准与知识、组织权限和平台设置统一到一个平台中，减少交付资料分散、审批链路不清和标准文件难查的问题。

## 主要用户

- 系统管理员：维护用户、角色、权限、全局配置和平台基础数据。
- 交付负责人：查看交付风险、项目进度、文档状态和项目复盘。
- 项目经理：维护项目台账、项目档案、文件上传、文件审核和团队协同。
- 电气工程师、软件工程师等专业人员：上传专业资料，查阅知识库和模板。
- 采购、财务、HSE：维护各自负责的项目资料、审批记录和风险记录。
- 标准管理员：维护档案模版、标准库和知识库。

权限种子还定义了国家负责人、专业负责人、外部合作方、只读用户和审计人员。角色只提供权限集合，业务接口不以角色名建立第二套白名单。

## 当前产品规则

- 知识库分类只保留一级分类，二级信息写入文件标题、备注或简介。
- 平台内常规文件入口统一在当前页面打开接近全视口的预览弹窗，不跳转到独立预览页，不重复展示文件信息栏。
- 知识库和文档模板点击文件标题进入统一弹窗预览。
- 知识库和文档模板列表展示预览热度和下载热度。
- 项目档案固定为文件夹/文件项两级，不按项目阶段生成目录；项目阶段是项目属性。
- 档案项按模版配置决定是否审核；需审核的新版本只有通过后才成为当前有效版本，免审项上传后直接生效。
- 审批人需要能查看新旧文件差异或内容变化说明。
- 项目概览筛选保持简洁，由统计卡、统一视图选择器、关键词搜索、刷新和创建操作组成。
- 项目概览默认展示“我的项目”，统一视图选择器提供“我的项目”“全部项目”“归档项目”；正常项目与归档项目分别查询和展示。
- 项目概览使用 Arco Pro 紧凑直角工作台样式，统计卡、附着式工具栏和容器内滚动表格形成连续的信息工作区。
- 项目分类由项目类型、合同类型、产品和关键词组成；团队负责人只设置销售、项目经理、电气和软件负责人。
- 项目阶段、进度百分比和预计/实际验收时间由统一进度命令维护，普通编辑只处理常规字段。
- 正常项目列表不放置操作列，查看、编辑、进度和归档集中在 800px 居中详情弹窗；归档列表只提供查看、恢复和受限永久删除。
- 项目、档案模版、标准和知识均保留不可变版本历史；发布内容不能原地覆盖。
- 飞书和企业微信只用于通讯录同步与通知发送，Secret 不返回明文。

## 当前实施与验收计划

| 优先级 | 工作项 | 调整原因 | 预期收益 | 当前状态 | 验收标准 |
| ------ | ------ | -------- | -------- | -------- | -------- |
| P0 | 持续执行 29 个 migration 的真实依赖发布验收 | 项目、档案、文件、审核和 Secret 迁移属于同一发布事务，静态测试不能替代 MySQL、Redis、MinIO 和 Worker 验证 | 降低数据迁移、对象存储引用和异步任务在发布窗口中的失败风险 | 本地固定镜像真实验收已通过；每次目标提交仍由 `integration` 与 `deploy` 重新执行 | migration、二次 seed、三组 migrator、API E2E、Worker 和浏览器关键流程全部通过，部署环境的 release id 与目标提交一致 |
| P0 | 建立文档事实自动检查 | 路由、字段、migration、测试数量和依赖版本会随源码变化，人工维护容易产生失配 | 让文档漂移在合并前被发现，减少交接和验收误判 | 已实现并接入本地质量脚本与 CI | CI 校验 Markdown 链接、源码路径、migration 数量、路由与权限规模和直接依赖清单 |
| P1 | 统一项目配置数据源 | 项目类型、合同类型、产品和关键词需要保持一致的配置边界 | 业务人员可通过配置调整选项，前后端不会出现取值不一致 | 已实现数据库字典聚合接口、服务端终检和前端统一消费 | API 返回启用项及稳定编码；服务端校验配置值；前端不维护独立业务枚举，只保留颜色展示映射 |
| P1 | 扩充权限、数据范围与审计契约测试 | 当前有 165 个 HTTP 路由和 87 个权限码，高风险动作还包含 Service 级业务终检 | 降低新增接口漏权限、漏数据范围或漏审计的风险 | 已覆盖 Controller 边界、角色矩阵、数据范围以及审计成功和失败路径 | Controller 权限覆盖、角色矩阵、敏感字段裁剪、成功/失败审计和超级管理员业务限制均有自动化断言 |
| P1 | 完成核心页面国际化收口 | 项目页面的校验提示、确认文案和成功消息需要统一管理 | 中英文体验一致，减少文案散落和后续改版成本 | 项目列表、详情和编辑抽屉已收口并增加防回退契约 | 核心项目页面固定文案全部进入中英文同构 key；测试阻止新增硬编码业务文案 |
| P1 | 固化项目概览 Arco Pro 工作台规范 | 项目经理需要在单一高密度工作区完成范围切换、统计筛选、检索和项目定位，页面还需要与公司最新设计规范保持一致 | 降低视线跳转和操作路径长度，提升宽表扫描效率，并为其他台账类页面提供可复用的密度、边框和交互基准 | Stitch“项目概览 - Arco Pro 直角精调版”已落地，真实服务浏览器验证、类型检查和生产构建已通过 | 三种视图可切换且 URL 状态可恢复；四张统计卡可筛选；工具栏单行附着表格；真实 NestJS、MySQL、Redis 和 MinIO 环境无控制台错误 |
