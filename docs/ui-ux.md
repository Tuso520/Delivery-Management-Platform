# 界面体验与 Arco Design 规范

## 当前标准

平台统一使用 Arco Design Vue 作为主要 UI 组件库。页面设计以企业内部工作台为目标，强调清晰、紧凑、稳定和可重复操作。

## 当前实现

- 已安装 `@arco-design/web-vue@2.58.0`，并从 `delivery-platform-web/src/main.ts` 引入。
- 全部运行页面使用原生 Arco Design Vue；旧 `arco-compat/`、旧图标别名、`@ts-nocheck` 兼容文件和退出运行时的页面样式已删除。
- `components/business/` 统一页面容器、工具栏、卡片、表格、抽屉、弹窗、状态、表单、空态、错误态和权限态。
- 页面模式、组件复用和完整路由见 [前端页面架构](frontend-architecture.md)，正式约束见 [前端实施规范](frontend-architecture-refactored.md)。

## 组件维护优先级

维护页面时，优先复用和检查以下组件：

- 按钮
- 表单
- 输入框
- 选择框
- 表格
- 弹窗
- 抽屉
- 分页
- 消息提示
- 通知提醒

## 验收要求

- `pnpm --dir delivery-platform-web build` 通过。
- 关键页面没有明显视觉回退。
- 表单校验、分页、弹窗确认和错误反馈正常。
- 加载、空数据、错误、禁用、只读、无权限和校验失败状态都经过检查。
- 不新增其他 UI 组件库。
- 中文与英文固定文案 key 集合一致，核心页面模板不保留硬编码中文。
- 浅色、深色和跟随系统模式都需检查；移动端重点保证查看、搜索、上传、审核和简单操作。

## 布局规则

- 业务页面保持紧凑、偏工作台风格。
- 筛选项尽量少，并优先保持在一行。
- 避免卡片套卡片和过大的装饰区域。
- 列表较长时优先在内部框架滚动，避免外层页面滚动导致界面不稳定。
- 项目概览采用 Arco Pro 紧凑工作台布局：24px 页面边距、4/8px 间距节奏、2px 圆角、`#E5E6EB` 弱边框和 `#F7F8FA` 表头背景。
- 项目概览四张统计卡横向排列，选中卡使用浅蓝底和顶部状态色边框；工具栏直接附着表格，搜索输入框与主按钮连续拼接。
- 项目概览表格不使用斑马纹，行悬停使用浅蓝高亮，首列固定，分页固定在列表工作区底部；窄屏下统计卡切换为两列。
- 项目概览字体优先使用 Inter 与苹方，数字使用等宽数字特性，按钮、输入框、选择器和标签统一保持 2px 圆角。
- 项目详情使用 800px 居中弹窗和半透明遮罩，弹窗最高 85vh；顶栏在同一行展示标题、权限动作和关闭按钮，正文使用 24px 水平边距并独立滚动。
- 项目详情不使用卡片嵌套，基本信息采用三列，阶段合同与团队采用四列，风险档案采用三列；分区之间只使用弱分隔线。
- 知识库和文档模板的文件标题预览交互保持一致。
