# 本地质量门禁

## 固定环境

- Node 20
- pnpm 10.34.4
- 后端：NestJS、Prisma 5.22.0、Jest
- 前端：Vue 3、Vite、vue-tsc、Vitest

如果 `pnpm` 不可用，先安装项目固定版本：

```powershell
npm install --global pnpm@10.34.4 --registry=https://registry.npmmirror.com
```

## 一键检查

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\quality-check.ps1
```

只验证代码、不重新安装依赖：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\quality-check.ps1 -SkipInstall
```

只跑类型检查和测试、不构建：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\quality-check.ps1 -SkipInstall -SkipBuild
```

## 手动检查命令

后端：

```powershell
cd delivery-platform-server
pnpm install --frozen-lockfile --prefer-offline
pnpm prisma:generate
pnpm type-check
pnpm test -- --runInBand
pnpm build
```

前端：

```powershell
cd delivery-platform-web
pnpm install --frozen-lockfile --prefer-offline
pnpm type-check
pnpm test
pnpm build
```

## 交付规则

- 任一命令失败，不允许生成正式发布包。
- 生产数据库结构变更必须使用 `prisma migrate deploy`，禁止使用 `prisma db push --accept-data-loss`。
- 每次发布必须保留类型检查、测试、构建和 Docker 健康检查结果。
