# 数据与权限架构

## 数据域

当前 Prisma Schema 包含 57 个模型，按以下领域组织：

| 领域 | 主要模型 |
|---|---|
| 用户权限 | User、Role、Permission、UserRole、RolePermission |
| 项目 | Project、ProjectMember、ProjectPayment、ProjectProcessRecord |
| 档案文件 | ArchiveTemplate、ArchiveTemplateItem、ProjectArchiveItem、File、FileReview、Attachment |
| 标准模板 | ChecklistTemplate、ChecklistTemplateItem、WorkflowCategory、WorkflowDocument、DocumentTemplate、DocumentTemplateVersion |
| 知识工具 | KnowledgeCategory、KnowledgeArticle、KnowledgeArticleVersion、ToolCategory、ToolItem |
| 全球配置 | Country、Currency、ExchangeRate、Language、Translation |
| 团队运营 | DailyReport、OkrObjective、KeyResult、PerformanceScore、SkillDefinition、SkillAssessment、TrainingPlan、TrainingParticipant |
| 审批复盘 | ApprovalTemplate、ApprovalStep、ApprovalTask、ApprovalAction、ProjectRetrospective、RetrospectiveAction |
| 系统运营 | Notification、NotificationRule、OperationLog、SystemConfig、DashboardWidget、IntegrationConfig、ApiKey、BackupRecord |
| 组织字典 | Department、DictionaryCategory、DictionaryItem |

数据库命名使用 snake_case，Prisma 字段使用 camelCase 并通过 `@map` 映射。

## 核心规则

- Project、User、File、Attachment、知识和模板等核心对象使用软删除。
- 项目金额同时保存原币、汇率、折算币种、折算金额、汇率日期和来源。
- 项目回款保存计划金额、到账金额和折算金额。
- 文件内容存入 MinIO，数据库保存对象路径、大小、MIME、上传人和业务归属。
- 种子脚本使用 upsert 或缺失补充，不覆盖服务器已有业务内容。

## 权限模型

权限编码格式为 `resource:action`，例如：

- `project:view`
- `attachment:upload`
- `template:download`
- `okr:score`
- `system:manage_backup`

角色权限页面按以下动作组展示：

- 查看
- 下载
- 上传
- 操作

`SUPER_ADMIN` 具备全局放行能力。普通用户除接口权限外，还必须通过项目成员关系或被授权的数据范围才能访问项目数据。

## 审计

全局审计拦截器记录已认证用户的 POST、PUT、PATCH 和 DELETE 操作。以下操作还需业务服务记录详细日志：

- 文件和附件下载
- 权限和角色变更
- 敏感配置访问
- 备份创建与下载
- 审批决定

日志写入 `operation_logs`，请求中的密码、Token、Secret 和 API Key 会脱敏。

## 附件归属

通用附件通过 `ownerType + ownerId` 关联知识、培训、项目过程记录、复盘等业务对象。附件可选关联项目，用于复用项目数据范围校验。
