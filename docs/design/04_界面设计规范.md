# 交付管理平台界面设计规范

## 设计目标

平台面向跨国家交付团队和管理人员，界面采用克制、清晰、可持续使用的企业后台风格。视觉升级不改变现有业务路由、接口、权限模型和数据结构。

## 视觉方向

- 主色：深松绿色，用于主操作、选中状态和关键数据。
- 中性色：暖灰白页面、纯白工作面、炭黑文字，避免大面积深蓝和高饱和渐变。
- 状态色：绿色表示完成，琥珀色表示提醒，红色表示风险，灰色表示未开始或停用。
- 圆角：页面工具和卡片不超过 8px，表单控件为 6px。
- 阴影：只用于弹窗和悬浮层；普通业务分区主要依靠边框和间距建立层级。
- 字体：优先使用 `IBM Plex Sans`、`Noto Sans SC`、`PingFang SC` 和系统无衬线字体。

## 页面结构

1. 登录页使用品牌区和登录工作区组成的双栏布局，移动端折叠为单栏。
2. 主框架由固定侧栏、紧凑顶栏和可滚动内容区组成。
3. 列表页按“筛选区、操作栏、数据区、分页”排列。
4. 详情页按信息主题分区，不使用多层卡片嵌套。
5. 表单页限制正文宽度，长表单按业务主题分段。
6. 业务页面最大宽度为 1680px，宽屏保持可扫描性，窄屏允许表格横向滚动。

## 交互规范

- 图标按钮必须具有可访问名称或工具提示。
- 主操作每个区域只保留一个实心按钮，其余使用普通或文字按钮。
- 表格行悬停、按钮按下、输入聚焦均提供明确反馈。
- 加载、空数据和错误状态使用统一的 Element Plus 状态组件。
- 手机端侧栏使用抽屉，筛选表单改为纵向排列，弹窗宽度不超过视口。
- 所有输入、按钮和导航保留键盘焦点样式。

## 组件边界

- `BasicLayout.vue`：负责侧栏、顶栏和内容区编排。
- `AppSidebar.vue`：负责品牌、权限菜单、折叠和移动端导航。
- `AppHeader.vue`：负责页面标题、上下文、语言和用户操作。
- `global.scss`：负责令牌、Element Plus 主题和既有页面结构兼容层。
- 页面组件继续负责业务状态、接口调用和局部结构，不重复实现全局视觉规则。

## 验收标准

- 登录页和后台页面视觉语言一致。
- 所有一级、二级菜单页面具有一致的间距、表格、表单、弹窗和空状态。
- 1366px、1920px 和移动端宽度下不存在内容遮挡或横向布局错位。
- 不新增 UI 依赖，不改变业务 API，不引入远程字体或运行时外部资源。
