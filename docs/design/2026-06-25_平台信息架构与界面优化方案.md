# 平台信息架构与界面优化方案

## 目标

本轮在不更换 Vue、Element Plus、NestJS、Prisma 和现有 API 架构的前提下，完成项目看板、项目台账、项目档案、工时报告、知识库、目标绩效、技能评估、系统配置和接口集成的统一调整。

## 设计原则

- 使用 Arco Design 的蓝色主色和中性色语义，界面保持扁平、清晰、低阴影。
- 页面顶栏只保留全局标题，业务页不再重复大标题和说明。
- 筛选、视图切换和主要操作放在同一工具栏；选择器保证能显示完整业务名称。
- 表格按信息优先级分配列宽，日期、状态、金额和操作不换行，长文本使用省略提示。
- 项目编号继续作为唯一业务标识，但默认不在项目台账列表展示。

## 业务调整

- 看板顺序调整为回款概览、阶段与状态、我的项目、最近创建。
- 项目台账展示折算人民币和项目经理、电气负责人、软件负责人。
- 档案目录按父子结构展示，统计仅计算可执行的叶子档案项；项目过程记录合并到项目档案。
- 工时报告统一支持日报、周报、月报，并提供八项工时统计。
- 知识分类改为单层业务分类，正文和附件按分类连续浏览。
- 目标分为年度 KPI、年度 OKR、季度 OKR、月度 KPI；指标权重先于评分配置，评分按评分人顺序逐级处理。
- 技能库按项目管理、电气、软件和通用能力建立，评估记录保存自评、上级评分和最终评分。
- 系统固定参数进入配置中心；公开配置仅暴露平台名称、登录文案等允许匿名读取的键。
- 接口集成增加飞书配置。

## 数据安全

- 种子数据采用幂等 upsert，不删除服务器业务数据。
- 系统配置种子不覆盖管理员已经修改的值。
- 评分审批、配置修改、文件操作继续经过后端权限和操作日志。
