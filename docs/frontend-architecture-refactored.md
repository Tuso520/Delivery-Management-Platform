# 交付管理平台前端实施规范

## 文档定位

本文记录交付管理平台前端当前的**正式信息架构、页面结构、组件体系、权限规则、文件能力和工程边界**。实现细节与完整路由表以 `frontend-architecture.md` 为准。

- 实现清单：`frontend-architecture.md`
- 实现基线：2026-07-16
- 适用范围：`delivery-platform-web/`
- 技术基线：Vue 3、TypeScript、Vite、Pinia、Vue Router、Arco Design Vue
- 视觉与工程模式：采用 Arco Design Vue 工作台式布局和业务组件体系
- 本文性质：当前架构规范；后续改动必须同步更新源码、本文和实现清单

---

# 1. 总体目标

前端遵循以下目标：

1. 统一应用壳、页面容器和业务组件。
2. 路由作为菜单、标题、权限和页面入口的唯一配置源。
3. 主导航仅保留工作台、项目管理、标准与知识三大业务域。
4. 设置从左侧菜单移至右上角齿轮入口。
5. 创建、查看、编辑统一使用弹窗或抽屉，不再保留独立详情页面形态。
6. 所有页面内容区使用全宽布局。
7. 服务端数据统一交由 TanStack Query 管理。
8. 权限统一使用权限码，数据范围由后端控制。
9. 文件预览统一为一个入口，Office 仅支持只读预览。
10. 保持 Arco Design Pro 风格的工作台密度，页面只使用当前业务组件和 Arco Design Vue。

# 2. 应用壳、布局与页面容器

## 2.1 顶部栏

顶部栏统一高度建议为 `48px`，保留：

- 菜单折叠按钮
- 当前页面标题
- 设置齿轮图标
- 中英文切换
- 浅色 / 深色 / 跟随系统切换
- 用户菜单

删除：

- 页面说明
- 面包屑
- 重复的页面级标题

页面标题直接读取路由元数据；详情对象名称可在数据加载后动态替换标题。

## 2.2 侧栏

统一尺寸：

- 展开：`240px`
- 收起：`64px`

菜单使用单组展开模式；侧栏折叠由顶栏和移动端交互控制。当前不持久化菜单展开组，避免把导航状态复制成第二份配置。

## 2.3 页面内容区

所有业务页面使用全宽内容区，不设置统一 `max-width`。

统一边距：

- 大屏：24px
- 桌面：20px
- 平板：16px
- 手机：12px

## 2.4 页面容器

建立：

- `PageContainer`
- `PageToolbar`
- `SectionCard`
- `StatCard`
- `StickyActionBar`

长表单使用固定底部操作栏，普通短表单不固定。

## 2.5 响应式

- 桌面端：完整功能
- 平板端：侧栏收起、表单两列、工具栏换行、表格横向滚动
- 手机端：重点保证查看、搜索、上传、审核和简单操作
- 复杂模板编辑、系统配置等可提示使用桌面端

# 4. 组件体系

## 4.1 技术策略

保留 `@arco-design/web-vue`，参考 `arco-design-pro-vue` 的布局、主题和页面模式，在现有项目中迁移。

不直接将现有项目替换为官方模板项目。

## 4.2 组件分层

### 平台通用组件

- `PageContainer`
- `PageToolbar`
- `SectionCard`
- `StatCard`
- `BusinessTable`
- `BusinessDrawer`
- `BusinessModal`
- `StatusBadge`
- `FileTypeBadge`
- `FormSection`
- `FormGrid`
- `ReadonlyField`
- `LoadingState`
- `EmptyState`
- `ErrorState`
- `PermissionDeniedState`
- `Can`

### 业务组件

按业务域放入 `features/` 或对应业务目录，例如：

- `ProjectDrawer`
- `ProjectOverviewCards`
- `ProjectArchiveTree`
- `FileReviewDrawer`
- `ArchiveTemplateEditor`
- `StandardDrawer`
- `KnowledgeDrawer`

## 4.3 BusinessTable

统一包装：

- 默认 20 条批次与容器内无限滚动
- loading
- 空态
- 错误态
- 横向滚动
- 固定操作列
- 行选择
- 表格密度
- 列宽和列显隐

主列表默认使用 large 行高，目录内嵌表格使用 medium 行高，弹窗和仪表盘中的辅助表格可使用 small。`BusinessTable` 通过容器宽度决定列布局：列宽合计未超过容器时移除显式列宽并均匀铺满，超过容器时保留业务列宽并在单一视口内横向滚动；所有表头和单元格禁止换行。

当前组件同时支持两种列声明：

- `columns` 数组配合具名 cell slot；
- 页面内 `a-table-column` 声明式列，包括 `v-for` 动态列和历史 `#columns` 具名列槽。

声明式列由 `BusinessTable` 转换为 Arco 的 `TableColumnData`，并规范化 `data-index`、`min-width` 等 kebab-case 属性。不得直接在页面新增声明式 `a-table`，避免当前 Vue/Arco 组合下列注册丢失；新增表格统一使用 `BusinessTable` 或显式 `columns` 数组。

操作列规则：

- 一般页面：主要操作直接显示，低频操作进入“更多”
- 正常项目概览不设操作列，点击固定在左侧的项目名称进入居中详情弹窗
- 项目归档列表仅显示查看、恢复、永久删除操作

## 4.4 抽屉

统一宽度级别：

- 560px
- 720px
- 960px
- 80vw

同一业务抽屉通过 `mode=create|view|edit` 复用。

## 4.5 权限展示

- 无权限操作默认隐藏
- 业务状态不允许时，显示禁用状态和原因

## 4.6 状态组件

建立按业务域分类的统一状态字典：

- project
- file
- review
- risk
- template
- task
- user

禁止页面自行定义状态文本和颜色。

# 6. 项目管理

## 6.1 项目概览

原“项目台账”改名为“项目概览”。

### 顶部统计

- 项目总数
- 进行中
- 已验收
- 高风险

统计卡可点击并筛选列表。

### 主筛选

左侧依次为“我的项目 / 全部项目”、关键词搜索框和搜索按钮，默认“我的项目”。“我的项目”包含当前用户担任销售、项目经理、电气负责人、软件负责人或普通项目成员的项目。关键词搜索：

- 项目名称
- 项目简称
- 项目编号
- 客户名称

右侧：

- 刷新
- 创建项目

### 列表字段

- 项目名称
- 区域：国家 / 城市
- 项目分类：项目类型 + 合同类型
- 当前阶段
- 项目进度
- 签约时间
- 验收时间：未验收时显示预计验收时间；无预计时间时显示“—”
- 合同金额：显示原币种金额
- 折算人民币
- 销售
- 项目经理

列表不显示项目编号和操作列。项目名称优先使用简称，单行省略并提供 Tooltip；名称固定在左侧，金额右对齐，表格允许横向滚动。

### 项目状态

- 草稿
- 进行中
- 暂停
- 已完成
- 已取消

### 项目阶段

固定阶段：

- 启动
- 深化
- 采购
- 施工
- 调试
- 测试
- 内验
- 外验
- 维保

列表只展示阶段标签，不允许行内修改。具备 `project:progress:update` 权限的人员在项目详情通过统一“修改进度”命令同时维护阶段、进度百分比和预计/实际验收时间。

阶段回退必须填写原因，并记录审计日志。

### 创建与编辑项目

创建和编辑共用 `ProjectDrawer(mode=create|edit)`，由 `BusinessDrawer` 提供 80vw 宽度，并使用 `FormSection`、`FormGrid`、`ReadonlyField` 和 `StickyActionBar`。响应式表单在大屏、桌面、平板和手机分别使用 4/3/2/1 列。分区包含：

- 基础信息
- 项目分类
- 合同与金额
- 计划阶段与进度
- 项目团队
- 风险与备注
- 档案模版

项目团队只保留销售、项目经理、电气负责人和软件负责人。项目类型、合同类型、产品和项目关键词通过项目聚合配置接口从平台字典加载；标签颜色由前端统一计算。创建模式支持保存草稿。

### 项目详情与归档

项目详情使用 800px 居中弹窗，最高为视口高度的 85%，正文独立滚动。弹窗顶栏按后端返回权限显示“编辑项目”“修改进度”“归档项目”，并提供关闭按钮；正文扁平展示项目基本信息、阶段/计划/合同、项目团队和风险/档案四个分区。归档最终权限由后端返回的 `canArchive` 决定。

归档项目通过项目概览右侧入口查看。归档列表显示项目名称、区域、项目类型、合同类型、归档人、归档时间、项目经理及查看/恢复/永久删除操作。永久删除只对超级管理员开放，并执行两次确认。

### 项目删除

- 正常项目详情不提供永久删除
- 超级管理员只能从归档列表永久删除已归档项目
- 存在文件、审核、财务或必须保留的审计关联时，服务端拒绝物理删除
- 成功和失败的高风险删除均记录审计日志

- 视频：浏览器原生格式优先，不支持时异步转码
- 音频：HTML5 Audio
- 不支持格式：提示并提供下载

全平台无 Office 在线编辑权限，不显示编辑入口。

# 9. 状态管理与请求架构

## 9.1 Pinia

仅保存：

- 用户
- 权限
- 主题
- 侧栏和菜单偏好
- 语言

## 9.2 TanStack Query

所有服务端数据使用 `@tanstack/vue-query`：

- 项目
- 档案
- 审核
- 模版
- 标准
- 知识
- 用户和设置

负责：

- 缓存
- 去重
- loading
- error
- 重试
- 失效
- 刷新

## 9.3 URL 状态

筛选和排序写入 URL query。列表不显示分页器，默认请求或显示 20 条，在容器接近底部时自动续载下一批；达到服务端 `total` 或返回不足一批时停止请求。

## 9.4 重试

- 查询请求有限重试
- 提交请求不自动重试
- 401、403 和业务校验失败不重试

## 9.5 错误分层

- 请求层：网络、401、服务不可用
- 页面层：页面加载失败、空态、权限不足
- 操作层：创建、保存、上传、审核等动作结果

# 10. 认证、权限与数据范围

## 10.1 权限模型

统一采用权限码，不直接判断角色名称。

四层权限：

- 登录权限
- 页面权限
- 操作权限
- 数据范围

数据范围由后端过滤，前端不得获取全部数据后自行隐藏。

## 10.2 Token

- Access Token：短期令牌
- Refresh Token：HttpOnly + Secure Cookie
- 多个并发 401 只触发一次刷新
- 刷新失败后退出登录

## 10.3 文件审核权限

所有人可见文件审核菜单。

- 普通用户：查看与自己相关的审核进度
- 审核人：执行当前审核任务
- 管理员：按权限查看全部审核任务

## 10.4 设置权限

所有人都能看见右上角设置齿轮图标。

具备对应操作权限的用户才可以执行设置修改；无权限项目默认只读或隐藏具体操作按钮。

## 10.5 超级管理员

保留 `SUPER_ADMIN`，但所有高风险操作必须审计。

# 12. 视觉、主题与国际化

## 12.1 主题

支持：

- 浅色
- 深色
- 跟随系统

默认跟随系统。

## 12.2 视觉密度

采用标准企业后台密度：

- 小圆角
- 浅边框
- 少阴影
- 高信息密度
- 不使用营销型大标题和重装饰

## 12.3 国际化

界面语言只保留：

- 中文
- 英文

主导航、六类设置及主要业务流程使用 `vue-i18n`，中英文 key 集合由测试保证同构。项目表单校验、确认与结果提示及部分组织权限深链文案仍为中文固定文案，列入当前实施计划。

业务数据不自动翻译。

语言状态由 `localeStore` 统一管理。

## 12.4 兼容边界

运行时源码不得包含：

- `arco-compat/`
- Element 风格 CSS
- 非 Arco 图标别名
- `@ts-nocheck` 兼容文件

## 12.5 TypeScript

当前编译门禁：

- `strict = true`
- `strictTemplates = true`
- `noUnusedLocals = true`
- `noUnusedParameters = true`

上述四项均已开启，并纳入 `pnpm --dir delivery-platform-web type-check`。

# 14. 实施结果与持续门禁

| 范围 | 当前结果 |
| ---- | -------- |
| 应用壳与导航 | 48px 顶栏、240/64px 侧栏、三大主导航、齿轮设置入口；路由注册表生成菜单、标题和权限 |
| 页面交互 | 项目、审核、标准、知识和档案模版均支持可分享深链并在列表页打开抽屉；筛选保留在 URL，批次页码由滚动续载管理 |

| 服务端状态 | 所有在用业务与组织设置页面接入 TanStack Query；Pinia 只保存会话、权限、外观和语言 |
| 权限 | 菜单、路由、动作按权限码；`SUPER_ADMIN` 仅前端展示绕过，后端仍做最终校验和审计 |

| 国际化 | 核心业务与设置中英文 key 同构；业务数据不翻译，隐藏组织权限深链仍以中文为主 |
| 自动化 | Vitest、Vue Test Utils、Query/i18n/路由/流程契约测试和真实 API Playwright 冒烟 |
