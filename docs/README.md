## Document Ownership

- Product and workflow changes: update `product.md`.
- Frontend, backend, data or permission changes: update `architecture.md` and `development.md`.
- Deployment, migration or server changes: update `deployment.md`.
- Validation results and manual browser checks: update `testing.md`.
- Security, backup, access or secret handling: update `security.md`.
- User-visible release changes: update `release.md` and root `CHANGELOG.md`.
