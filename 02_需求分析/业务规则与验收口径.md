# 业务规则与验收口径

## 通用规则

- 所有核心数据默认软删除。
- 所有敏感操作写入操作日志。
- 金额字段保留原币种和折算信息。
- 文件存储在 MinIO，数据库保存文件索引、版本、业务关联和预览元数据。
- API 返回统一结构，前端只消费 `data` 内业务对象。

## 权限规则

- 未登录访问业务接口返回 401。
- 已登录但无权限访问返回 403。
- 用户角色和权限变更必须即时影响新请求。
- 前端菜单展示不能替代后端权限校验。

## 知识库验收

| 验收项 | 标准 |
| --- | --- |
| 分类重复 | 同一父级下重复分类被阻止并提示 |
| 文章列表 | 可按分类、关键字、状态筛选 |
| 文件上传 | 支持 doc、docx、xls、xlsx、ppt、pptx、pdf、png、jpg |
| 在线预览 | 不同文件类型进入对应预览布局 |
| 预览失败 | 显示失败原因和下载入口 |
| 权限 | 无权限用户不能新增、编辑、删除或下载 |

## 部署迁移验收

- 服务器已有 `.env` 不被覆盖。
- MySQL、Redis、MinIO 持久化卷不被删除。
- 部署前自动备份数据库和上传文件。
- 迁移失败时可回滚到上一版本镜像或 Git commit。
- 部署完成后健康检查通过。
