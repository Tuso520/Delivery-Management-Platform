# Arco Design 迁移检查记录

## 检查结论

当前前端工程以 `@arco-design/web-vue` 作为主 UI 组件库，`package.json` 中已不依赖旧组件库。页面中的旧组件、旧提示 API 和旧标签已清理；后续新增页面必须继续使用 Arco 组件。

## 已检查内容

- 前端依赖：确认存在 `@arco-design/web-vue`。
- 请求层：`delivery-platform-web/src/api/request.ts` 使用 Arco `Message`。
- 路由菜单：当前页面按工作台、项目管理、标准与知识、绩效与团队、组织与权限、系统设置组织。
- 乱码修复：已修复若干选择框占位、表格标题、按钮和标签中的乱码文案。
- 知识库预览：本地模拟服务提供 doc、xlsx、ppt、pdf、图片预览样例。

## 继续约束

- 禁止重新引入旧组件库。
- 禁止新增乱码文案。
- 组件替换后必须跑前端类型检查、测试和构建。
- 涉及页面功能时必须使用本地模拟服务做登录和页面访问验证。
