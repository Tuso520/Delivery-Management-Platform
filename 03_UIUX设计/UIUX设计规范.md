# UIUX 设计规范

## 整体风格

- 主组件库：Arco Design Vue。
- 风格关键词：企业级、清晰、克制、高效、可扫描。
- 页面结构：左侧菜单、顶部用户区、主体内容区。
- 卡片使用：只用于独立信息组、统计块、表单容器和重复项，不做大面积装饰。
- 文案：所有标题、按钮、占位、表格列名必须为可读中文或配置化文案。

## 布局规范

- 列表页：筛选区、操作区、表格、分页。
- 详情页：基础信息、业务分组、关联文件、操作记录。
- 配置页：左侧分类或树结构，右侧表格或表单。
- 知识库：左侧分类树，中间文章列表或详情，右侧附件和预览入口。

## 组件规范

| 场景 | 组件 |
| --- | --- |
| 表格 | `a-table` |
| 表单 | `a-form`、`a-form-item` |
| 输入 | `a-input`、`a-input-number`、`a-textarea` |
| 选择 | `a-select`、`a-tree-select`、`a-cascader` |
| 日期 | `a-date-picker`、`a-range-picker` |
| 弹窗 | `a-modal` |
| 抽屉 | `a-drawer` |
| 上传 | `a-upload` 或封装 FileUploader |
| 消息 | `Message`、`Notification` |
| 确认 | `Modal.confirm` |

## 知识库预览规范

- PDF：使用嵌入式预览区域，保留下载入口。
- 图片：使用图片预览，支持缩放和适配容器。
- Word：展示在线转换后的页面或文本摘要，标明页数和版本。
- Excel：展示 Sheet 列表和表格预览，保留下载入口。
- PowerPoint：展示幻灯片列表或缩略内容，保留下载入口。
- 未知格式：展示文件信息、失败原因和下载按钮。

## 无障碍和异常

- 所有交互控件应有明确文字或图标语义。
- 加载、空数据、无权限、预览失败、网络失败必须有可见状态。
- 批量操作和删除操作必须二次确认。
